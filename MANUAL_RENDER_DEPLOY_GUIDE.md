# Render手动部署指南

## 🔍 问题描述

代码已本地修改但Render服务器未更新。当前部署版本仍返回占卜格式的回答（包含【卦象/签文】、【吉凶判断】等标题），不符合用户要求。

## ✅ 本地修改已验证

1. **系统提示已修改**：添加了"绝对不要使用占卜格式"要求
2. **占卜标题已排除**：系统提示明确禁止使用【卦象/签文】、【吉凶判断】等标题
3. **回答逻辑已修正**：所有问题都使用普通问答格式

## 🚀 手动部署步骤

### 方法1：通过Render控制台（推荐）

1. **访问Render控制台**
   - 打开浏览器访问：https://dashboard.render.com
   - 使用你的GitHub账号登录

2. **选择服务**
   - 在Dashboard中找到服务：`knowledge-base-qa`
   - 点击进入服务详情页

3. **手动触发部署**
   - 在服务详情页顶部找到 **"Manual Deploy"** 按钮
   - 点击下拉选择 **"Deploy latest commit"**
   - 等待部署开始（约2-3分钟）

4. **验证部署**
   - 部署完成后，测试API：
     ```bash
     curl -X POST -H "Content-Type: application/json" \
          -d '{"question":"什么是四柱预测学？"}' \
          https://knowledge-base-qa.onrender.com/api/qa/ask
     ```
   - 检查回答是否**不包含**占卜格式标题

### 方法2：通过GitHub Webhook（如果网络恢复）

1. **推送代码到GitHub**
   ```bash
   git push origin main
   ```

2. **Render会自动触发部署**

## 📊 部署验证检查清单

部署完成后，请测试以下问题验证回答格式：

1. ✅ **"什么是四柱预测学？"** - 应返回普通问答，无占卜格式
2. ✅ **"帮我算一下明天的运势"** - 应返回普通回答，解释占卜知识
3. ✅ **"预测一下我的工作前景"** - 应返回科普性解释，非实际占卜

**预期结果**：所有回答都应简洁、准确、专业，**绝不使用**：
- 【卦象/签文】
- 【吉凶判断】 
- 【解读分析】
- 【建议指引】

## ⚠️ 注意事项

1. **免费套餐限制**：Render免费服务有休眠机制，首次访问可能较慢
2. **内存存储**：服务器重启会清空向量存储，可能需要重建知识库索引
3. **监控**：部署后建议运行监控脚本检查系统状态

## 🆘 故障排除

如果部署后仍有问题：

1. **检查Render日志**
   - 在服务详情页点击"Logs"标签
   - 查看是否有构建或运行时错误

2. **验证环境变量**
   - 确保API密钥等环境变量正确配置

3. **测试健康检查**
   ```bash
   curl https://knowledge-base-qa.onrender.com/health
   ```
