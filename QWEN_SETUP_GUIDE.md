## 🔧 配置详情

### 当前配置（`.env`文件）
```env
# 通义千问 (阿里云) 大模型配置
NEXT_PUBLIC_LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
NEXT_PUBLIC_LLM_API_KEY=your_qwen_api_key_here  # ← 需要您替换为真实密钥
NEXT_PUBLIC_LLM_MODEL=qwen-turbo                # 可选：qwen-turbo, qwen-plus, qwen-max
```

### 通义千问模型选项

| 模型名称 | 特点 | 适用场景 | 价格（参考） |
|---------|------|---------|------------|
| **qwen-turbo** | 响应速度快，成本低 | 日常对话、简单问答 | 较低 |
| **qwen-plus** | 能力均衡，性价比高 | 复杂推理、代码生成 | 中等 |
| **qwen-max** | 能力最强，效果最好 | 专业分析、创意写作 | 较高 |
| **qwen2.5-32b-instruct** | 最新版本，能力全面 | 所有场景 | 中等偏高 |

## 🚀 快速开始

### 步骤1：获取通义千问API密钥

1. 访问 [阿里云百炼控制台](https://dashscope.console.aliyun.com/)
2. 注册/登录阿里云账号
3. 开通"通义千问"服务
4. 在"API密钥管理"中创建新的API密钥
5. 复制以 `sk-` 开头的API密钥

### 步骤2：更新配置文件

编辑 `.env` 文件，将以下行中的占位符替换为您的真实API密钥：

```env
NEXT_PUBLIC_LLM_API_KEY=sk-您的真实API密钥  # 替换这里
```

### 步骤3：验证配置

运行测试脚本验证配置：

```bash
node test-qwen-connection.js
```

### 步骤4：启动应用

```bash
npm run dev
```

访问 http://localhost:5173 开始使用通义千问大模型。

## 🎮 使用方式

### 方式一：通过UI界面配置（推荐）

1. 访问应用首页
2. 点击右上角设置按钮（⚙️图标）
3. 在"通义千问配置"面板中：
   - **API Base URL**: 已预设为通义千问端点
   - **通义千问API密钥**: 输入您的API密钥
   - **通义千问模型**: 选择模型（默认为qwen-turbo）
4. 点击"保存配置"

### 方式二：直接修改环境变量

如果您更喜欢命令行方式，可以直接编辑 `.env` 文件。

## 💡 使用技巧

### 1. 模型选择建议
- **日常使用**：qwen-turbo（速度快，成本低）
- **专业分析**：qwen-plus（能力均衡）
- **重要任务**：qwen-max（效果最好）

### 2. 成本控制
- 通义千问按token计费，注意控制使用量
- 可以从qwen-turbo开始试用，需要时再升级
- 定期检查阿里云控制台的用量和费用

### 3. 性能优化
- 对于占卜问答，qwen-turbo通常足够
- 如果响应慢，可以尝试切换到qwen-plus
- 复杂的多轮对话建议使用qwen-max

## ⚠️ 注意事项

### 1. API密钥安全
- 不要将API密钥提交到GitHub等公开仓库
- `.env` 文件已在 `.gitignore` 中排除
- 定期轮换API密钥

### 2. 服务限制
- 通义千问API有调用频率限制
- 免费额度有限，注意监控使用量
- 部分高级功能可能需要额外开通

### 3. 网络要求
- 确保网络可以访问阿里云服务
- 如有网络问题，可尝试使用代理
- 国内用户访问通义千问通常速度较快

## 📞 故障排除

### 常见问题

#### Q1: API密钥无效
- 检查密钥是否以 `sk-` 开头
- 确认已在阿里云控制台开通服务
- 验证密钥是否有足够的权限

#### Q2: 网络连接失败
- 检查网络连接
- 确认可以访问 `dashscope.aliyuncs.com`
- 尝试使用其他网络环境

#### Q3: 模型不可用
- 检查模型名称是否正确
- 确认您的账户有该模型的使用权限
- 尝试使用其他模型名称

#### Q4: 响应速度慢
- 尝试切换到 qwen-turbo 模型
- 检查网络延迟
- 减少单次请求的token数量
