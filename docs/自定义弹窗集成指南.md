# 自定义弹窗集成指南

## 概述

本文档介绍了如何在项目中集成自定义弹窗组件，以替换浏览器默认的`confirm`弹窗。自定义弹窗提供了更好的用户体验、现代化的UI设计和一致的视觉风格。

## 核心组件

### 1. 自定义确认弹窗 (`CustomConfirmDialog.tsx`)

位于：`/components/CustomConfirmDialog.tsx`

**功能特性：**
- 支持不同类型的弹窗（warning、info、danger）
- 响应式设计和动画效果
- ESC键和背景点击关闭
- 阻止背景滚动
- 无障碍支持

### 2. 弹窗状态管理Hook (`useConfirmDialog.ts`)

位于：`/lib/hooks/useConfirmDialog.ts`

**API说明：**
```typescript
interface ConfirmDialogOptions {
  title: string
  message: string
  confirmText?: string       // 默认："确认"
  cancelText?: string        // 默认："取消"
  type?: 'warning' | 'info' | 'danger'  // 默认："warning"
  onConfirm?: () => void
  onCancel?: () => void
}

// 使用示例
const {
  isOpen,
  options,
  showConfirm,
  closeDialog,
  handleConfirm,
  handleCancel
} = useConfirmDialog()

// 显示弹窗并等待用户确认
const confirmed = await showConfirm({
  title: '开始新对话',
  message: '您确定要开始新的对话吗？',
  confirmText: '开始新对话',
  cancelText: '取消',
  type: 'info'
})
```

## 集成步骤

### 步骤1：导入组件和Hook

```typescript
import CustomConfirmDialog from '@/components/CustomConfirmDialog'
import { useConfirmDialog } from '@/lib/hooks/useConfirmDialog'
```

### 步骤2：初始化弹窗状态

```typescript
const {
  isOpen,
  options,
  showConfirm,
  closeDialog,
  handleConfirm,
  handleCancel
} = useConfirmDialog()
```

### 步骤3：在组件中使用弹窗

**示例1：新建对话确认**
```typescript
const handleNewChat = async () => {
  const confirmed = await showConfirm({
    title: '开始新对话',
    message: '您确定要开始新的对话吗？当前对话记录将被清除。',
    confirmText: '开始新对话',
    cancelText: '取消',
    type: 'info',
    onConfirm: () => {
      // 确认后的操作
      setMessages([])
      setInput('')
    }
  })
}
```

**示例2：危险操作确认**
```typescript
const handleClearHistory = async () => {
  const confirmed = await showConfirm({
    title: '清空历史记录',
    message: '您确定要清空所有历史记录吗？此操作不可撤销。',
    confirmText: '清空记录',
    cancelText: '取消',
    type: 'danger',
    onConfirm: () => {
      // 确认后的操作
      setMessages([])
    }
  })
}
```

### 步骤4：渲染弹窗组件

```tsx
<CustomConfirmDialog
  isOpen={isOpen}
  onClose={closeDialog}
  onConfirm={handleConfirm}
  title={options.title}
  message={options.message}
  confirmText={options.confirmText}
  cancelText={options.cancelText}
  type={options.type}
/>
```

## 使用场景

### 1. 新建对话
```typescript
const handleNewChat = async () => {
  const confirmed = await showConfirm({
    title: '开始新对话',
    message: '您确定要开始新的对话吗？当前对话记录将被清除。',
    confirmText: '开始新对话',
    cancelText: '取消',
    type: 'info'
  })
  
  if (confirmed) {
    setMessages([])
    setInput('')
  }
}
```

### 2. 清空历史
```typescript
const handleClearHistory = async () => {
  const confirmed = await showConfirm({
    title: '清空历史记录',
    message: '您确定要清空所有历史记录吗？此操作不可撤销。',
    confirmText: '清空记录',
    cancelText: '取消',
    type: 'danger'
  })
  
  if (confirmed) {
    localStorage.removeItem('chatHistory')
    setMessages([])
  }
}
```

### 3. 重置表单
```typescript
const handleResetForm = async () => {

  if (confirmed) {
    setFormData(initialData)
  }
}
```

## 样式定制

### 颜色主题
自定义弹窗支持三种类型，每种类型有不同的颜色方案：

1. **信息类型 (info)**
   - 背景：`bg-blue-50`
   - 边框：`border-blue-200`
   - 图标：`text-blue-600`
   - 按钮：`bg-blue-600`

2. **警告类型 (warning)**
   - 背景：`bg-orange-50`
   - 边框：`border-orange-200`
   - 图标：`text-orange-600`
   - 按钮：`bg-orange-600`

3. **危险类型 (danger)**
   - 背景：`bg-red-50`
   - 边框：`border-red-200`
   - 图标：`text-red-600`
   - 按钮：`bg-red-600`

### 动画效果
弹窗包含以下动画：
- `animate-scale-in`：缩放进入动画
- 背景遮罩淡入效果

## 最佳实践

### 1. 确认文本
- 使用明确的动词描述操作（如"开始新对话"、"清空记录"）
- 保持简洁，不超过4个中文字符

### 2. 取消文本
- 默认使用"取消"
- 保持一致性

### 3. 消息内容
- 清晰地说明操作后果
- 对于危险操作，明确提示"此操作不可撤销"

### 4. 类型选择
- **信息类型**：一般性操作确认
- **警告类型**：需要用户特别注意的操作
- **危险类型**：可能导致数据丢失的破坏性操作

## 注意事项

1. **阻止默认行为**：在所有需要用户确认的操作中，使用自定义弹窗替换`window.confirm()`
2. **异步处理**：`showConfirm`返回Promise，确保正确处理异步流程
3. **状态管理**：弹窗状态由Hook管理，确保组件卸载时清理状态
4. **无障碍支持**：弹窗包含适当的ARIA标签和键盘导航支持

## 故障排除

### 常见问题

1. **弹窗不显示**
   - 检查`isOpen`状态是否为`true`
   - 确认`showConfirm`已被调用

2. **弹窗无法关闭**
   - 确保`onClose`回调函数正确设置
   - 检查是否有阻止事件冒泡的代码

3. **样式不正确**
   - 确认Tailwind配置包含必要的颜色类
   - 检查是否有CSS冲突
