# 🚀 10分钟免费部署指南

## 📋 准备工作（2分钟）

### 1. 注册账号（如已有可跳过）
- **GitHub**: https://github.com/signup （如已有代码仓库）
- **Railway**: https://railway.app/signup （后端部署）
- **Vercel**: https://vercel.com/signup （前端部署）

### 2. 准备API密钥
- 确保有有效的通义千问API密钥
- 备用：Tencent Cloud API密钥（可选）

## 🎯 核心步骤（8分钟）

### 第一步：上传代码到GitHub（3分钟）

```bash
# 1. 初始化仓库（如果还没做）
git init
git add .
git commit -m "准备免费部署"

# 2. 创建GitHub仓库
# 访问 https://github.com/new
# 输入仓库名：knowledge-base-qa
# 保持公开（免费账户需要）

# 3. 推送代码
git remote add origin https://github.com/你的用户名/knowledge-base-qa.git
git branch -M main
git push -u origin main
```

### 第二步：部署后端到Railway（3分钟）

1. **访问 Railway**: https://railway.app
2. **新建项目**：
   - 点击 "New Project" → "Deploy from GitHub repo"
   - 选择你的 `knowledge-base-qa` 仓库
3. **配置环境变量**（在Railway项目设置中）：
   ```
   NEXT_PUBLIC_LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
   NEXT_PUBLIC_LLM_API_KEY=你的通义千问API密钥
   NEXT_PUBLIC_LLM_MODEL=qwen-turbo
   PORT=3001
   NODE_ENV=production
   ```
4. **获取后端地址**：
   - 等待部署完成（约1-2分钟）
   - 记下分配的域名，如：`https://your-app.up.railway.app`

### 第三步：部署前端到Vercel（2分钟）

1. **访问 Vercel**: https://vercel.com
2. **导入项目**：
   - 点击 "New Project"
   - 从GitHub导入 `knowledge-base-qa` 仓库
3. **自动配置**（Vercel会自动检测）：
   - Framework: Vite
   - Build Command: `npm run build`
   - Output Directory: `dist`
4. **添加环境变量**：
   ```
   VITE_API_URL=你的Railway后端地址
   ```

## ⚠️ 注意事项

### Railway免费限制：
- **每月500小时**运行时间（约20天连续运行）
- **休眠机制**：无请求时自动休眠，首次访问有延迟

- 上传大文件可能失败
- 建议单个文件 < 10MB
